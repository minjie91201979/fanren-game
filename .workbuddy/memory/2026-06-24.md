## 灵药园/炼丹室/炼器室 Realm 枚举匹配 Bug 修复
- **根因**：`Realm` 枚举值为 `'QI_REFINING'` / `'FOUNDATION'` 等英文，但 `garden.ts` / `alchemy.ts` / `forging.ts` 的 `getAvailable*` 函数用中文数组 `['炼气期', '筑基期', ...]` 做 `indexOf` 匹配，返回 -1 导致全部作物/配方被过滤，按钮禁用无法操作
- **修复**：3 个文件统一添加 `REALM_CN_ORDER` 映射表，`getAvailable*` 函数先将 `player.realm`（如 `'QI_REFINING'`）通过 `REALM_NAMES` 转为中文名（`'炼气期'`）再匹配
- **Garden.tsx 额外修复**：
  - `handlePlant` 改用 `usePlayerStore.getState()` 获取最新玩家数据（避免闭包陷阱）
  - `setPlants` 改用函数式更新 `setPlants(prev => ...)` 确保基于最新数组
  - `planting` 状态从 `CropData | null` 简化为 `boolean`
  - `useCallback` 依赖从 `[plants, player.gold, modifyGold]` 精简为 `[]`
- 构建通过：84 模块 0 错误
