# 凡人修仙传

> 水墨粒子风格修仙题材单机 RPG 游戏 · v0.1.0

基于 **React + Phaser.js + Zustand** 的回合制修仙单机游戏，以天南大陆为舞台，体验从凡人到化神的漫漫仙途。

---

## 技术栈

| 类别 | 技术 |
|:---|:---|
| 框架 | React 18 + TypeScript 5 |
| 游戏引擎 | Phaser.js 3.80（战斗 / 突破动画场景） |
| 状态管理 | Zustand 4 + Immer（不可变数据） |
| 事件总线 | Mitt 3（全局事件通信） |
| 构建工具 | Vite 5 |
| 包管理 | pnpm monorepo |
| 运行环境 | Electron 28（桌面客户端） |

---

## 项目结构

```
fanren-game-git/
├── packages/
│   ├── core/          # 核心包 — 类型 / 常量 / 工具函数（零依赖）
│   ├── data/          # 数据包 — Store / 事件总线 / 存档系统
│   ├── game/          # 引擎包 — Phaser 场景 / 实体 / 粒子特效
│   ├── ui/            # UI 包 — React 页面 / 组件 / 全局样式
│   └── app/           # 应用包 — Electron 入口 / 路由 / 扩展数据
└── assets/            # 静态资源（背景 / 精灵 / 图标 / 音效）
```

**app 包**是实际运行入口，内含 core/data/game/ui 全部代码的自包含副本，并额外扩展了技能数据库、物品数据库、地图数据等游戏内容。

| 筑基期 | 3 | 250 | 50% |

### 战斗系统

回合制战斗，支持：

- ⚔️ 普通攻击 / 🛡️ 防御 / 🏃 逃跑 / 🔥 释放技能
- MP 消耗 + 冷却（CD）机制
- 技能附加效果：灼烧、麻痹、减速、护盾、Buff/Debuff
- 敌方 AI 自动反击，低血量概率逃跑
- 自动战斗开关，状态持久化到 localStorage
- 胜利奖励：灵石 + 修为（技能击杀 80 / 普攻击杀 50）

### 宗门系统

黄枫谷提供四大设施：

- **藏经阁**：消耗贡献点兑换功法
- **演武场**：切磋比试，胜率随攻击力变化
- **丹药堂**：每月领取丹药
- **任务堂**：接取/提交宗门任务（带进度条），获得灵石和贡献点

### 洞府

核心枢纽，支持：

- 闭关修炼（消耗 2 时辰，获得修为）
- 洞府升级（Lv1→Lv5，修炼加成递增）
- 建筑建造：静修室 / 炼丹室 / 炼器室 / 藏宝阁 / 灵药园 / 阵法中心
- 建筑升级效果：修炼加成 / 炼丹成功率 / 灵药产量等

### 存档系统

- **6 个存档槽位**，任意创建/加载/删除
- **自动存档**：每 5 分钟自动保存（非菜单页面）
- **快速存档**：洞府一键保存 + Toast 提示
- 存档信息：角色名、境界、灵石、时间戳
- 数据存储于 `localStorage`

## 数据流

```
用户操作 → React 组件 → Zustand Store（zustand + immer）
                              │
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
               playerStore  caveStore  combatStore
                    │         │         │
                    └─────────┼─────────┘
                              ▼
                       localStorage（自动持久化）
```

- **状态变更**：通过 Store 的 `setState` / `getState` 实现
- **跨 Store 通信**：延迟引用 `useXxxStore.getState()` 避免循环依赖
- **页面间通信**：`window.dispatchEvent(new CustomEvent('navigate', ...))`
- **全局通知**：`window.dispatchEvent(new CustomEvent('notification', ...))`
- **自动保存**：AppRouter 监听页面切换，自动启停 5 分钟定时器

## 开发说明

### 代码组织

- `packages/app/src/` 是实际运行目录，包含所有业务代码
- `packages/core/` 等独立包需通过 monorepo 的路径别名引用
- TypeScript 严格模式，所有接口/枚举统一定义于 `core/src/types/index.ts`

### 命名约定

| 前缀 | 含义 | 示例 |
|:---|:---|:---|
| `useXxxStore` | Zustand Hook | `usePlayerStore` |
| `handleXxx` | 事件处理器 | `handleStartCombat` |
| `ITEM_DB` / `SKILL_DB` | 数据库常量 | `ITEM_DB`, `SKILL_DB` |

### 注意事项

- 不要在 Store 之间直接导入，使用 `getState()` 延迟引用避免循环依赖
- `Realm` 枚举值与字符串对应：`Realm.QI_REFINING` = "炼气期"
- 自动战斗状态通过 `localStorage` 持久化，键名为 `autoBattle`
- 存档数据亦存于 `localStorage`，键名格式为 `save_slot_{n}`
- Phaser 场景（CombatScene / WorldMapScene / BreakthroughScene）为早期独立实现，当前游戏以 React 页面为主
